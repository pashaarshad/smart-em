Smart Event Management System

A Synopsis Submitted for Major Project (MCA/B.Tech)

1. Abstract

The "Smart Event Management System" is a robust, web-based platform designed to streamline the organization of college fests. Traditional methods typically involve tedious paperwork and manual payment verification, leading to inefficiencies. Our proposed system addresses these challenges with a centralized digital solution featuring secure Google authentication, real-time registration, and novel Optical Character Recognition (OCR) for payment verification. Built with Next.js and Firebase, it ensures efficiency and scalability.

2. Introduction

College fests attract thousands of participants, requiring precision in management. This project digitizes the entire lifecycle from event browsing to registration. The core innovation is "Smart" verification using machine learning-based OCR and PDF parsing to automate payment reconciliation, eliminating manual bottlenecks.

3. Problem Statement

Current practices suffer from:

  • Manual Data Entry: Redundancy and errors from spreadsheets.
  • Verification Lag: Days spent cross-checking screenshots with bank statements.
  • Duplicate Registrations: Lack of real-time uniqueness checks.

4. Existing System

  • Google Forms: Basic data collection.
  • Manual Checks: Volunteers manually verifying transactions.
  • Paper Receipts: Prone to loss and forgery.

5. Limitations of Existing System

  • High human error rate.
  • Time-consuming processes.
  • Lack of instant confirmation.
  • Security concerns with shared data.

6. Proposed System

A comprehensive web app automating workflow:

  • Automated OCR: Scans screenshots to verify UTR numbers instantly.
  • Duplicate Prevention: Real-time database checks for UTR uniqueness.
  • Admin Auto-Verify: Parses bank statement PDFs to bulk approve registrations.
  • Role-Based Access: Secure environments for Students and Admins.

7. Objectives of the Project

  • Eliminate manual paperwork.
  • Achieve near-100% verification accuracy.
  • Provide a seamless user experience.
  • Empower organizers with real-time monitoring.

8. Scope of the Project

The system covers registration and verification for multiple event categories. It is modular, supporting future extensions like QR attendance and certificate generation.

9. System Architecture

Fig 1: System Architecture & Database Design

A serverless architecture using Next.js for the client (handling UI and OCR) and Firebase for backend services (Auth, Firestore, Storage).

10. Modules Description

1. Authentication & Registration

Secure Google OAuth log in. Multi-step registration forms with team details and OCR-backed payment upload.

2. Payment Verification Module

Client-side Tesseract.js processing for instant feedback. Server-side checks for data integrity.

3. Admin Dashboard

Tabulated views of data. "Auto Verify" tool using PDF.js to parse bank statements and update statuses.
